features

Everything you need

Built on IntersectionObserver. No scroll listeners. No layout thrashing. No dependencies.

🎯
Add Any Class
Use ss-class="active" to toggle any CSS class when an element enters the viewport. Bring your own CSS.
🎨
12 Built-in Animations
fade, fade-up/down/left/right, zoom, zoom-out, flip-x/y, blur, reveal-up, reveal-left. Zero extra CSS.
🗺️
Section Tracking
Automatically adds a class to nav links when their target section is in view. One attribute on your nav.
📊
Scroll Progress
Track how far the user has scrolled through an element as a 0–1 value. Animate progress bars, counters, anything.
🔁
Once or Repeat
Trigger once and stop observing, or replay on every enter/leave with ss-repeat.
⚡
Stagger Groups
Animate children in cascade with ss-group and ss-stagger="80". Automatic nth-child delays.
📞
JS Callbacks
Full programmatic API: onEnter, onLeave, onProgress hooks per element with complete control.
🔍
Custom Threshold & Margin
Per-element threshold (0–1) and rootMargin for pixel-perfect trigger points.
usage

Simple HTML attributes

Animate with built-in effects 
<!-- Built-in animation effect -->
<div ss-animate="fade-up">Fades up on scroll</div>

<!-- With delay (ms) -->
<div ss-animate="fade-left" ss-delay="200">Delayed fade</div>

<!-- Repeat every time it enters viewport -->
<div ss-animate="zoom" ss-repeat>Repeating zoom</div>

<!-- Custom threshold (0–1, how much must be visible) -->
<div ss-animate="blur" ss-threshold="0.5">Half visible to trigger</div>
Add any custom class 
<!-- Toggle any class when in viewport -->
<nav ss-class="scrolled">...</nav>
<section ss-class="active highlighted">...</section>

<!-- Remove class on leave (repeat mode) -->
<div ss-class="in-view" ss-repeat>Toggles class on/off</div>

<!-- Add class + also add to a different target -->
<section ss-class="active" ss-class-target="#sidebar">...</section>
Section tracking & stagger groups 
<!-- Stagger children with cascading delay -->
<ul ss-group="fade-up" ss-stagger="80">
  <li>First — 0ms delay</li>
  <li>Second — 80ms delay</li>
  <li>Third — 160ms delay</li>
</ul>

<!-- Auto active-class on nav links when section is in view -->
<nav ss-nav="nav-active">
  <a href="#about">About</a>
  <a href="#work">Work</a>
</nav>

<!-- Progress: fires onProgress(ratio) as user scrolls -->
<section ss-progress="my-progress">...</section>
JavaScript API 
// Initialize (auto-scans DOM for ss-* attributes)
const ss = ScrollSpark.init()

// Add trigger programmatically
ss.on('.hero', {
  animate: 'fade-up',
  delay: 200,
  threshold: 0.2,
  repeat: false,
  onEnter(el) { console.log('entered', el) },
  onLeave(el) { console.log('left', el) },
})

// Add custom class toggle
ss.addClass('section', 'in-view', { repeat: true })

// Stagger a group
ss.stagger('.card', { animate: 'fade-up', stagger: 100 })

// Track scroll progress through an element
ss.progress('#about', (ratio) => {
  progressBar.style.width = (ratio * 100) + '%'
})

// Track active nav section
ss.trackNav('nav', 'nav-active')

// Remove / destroy
ss.off('.hero')
ss.destroy()

api reference

HTML Attributes

AttributeTypeDefaultDescription
ss-animatestring—Built-in effect name: fade, fade-up, fade-down, fade-left, fade-right, zoom, zoom-out, flip-x, flip-y, blur, reveal-up, reveal-left
ss-classstring—One or more space-separated class names to add when element enters viewport. Removed on leave if ss-repeat.
ss-class-targetCSS selector—Add the class to this element instead of (or in addition to) the trigger element.
ss-delaynumber (ms)0Milliseconds to wait after entering viewport before activating.
ss-thresholdnumber 0–10.15Fraction of element visible before trigger fires.
ss-marginstring"0px"rootMargin for IntersectionObserver. E.g. "-100px 0px" to trigger before entering view.
ss-repeatbooleanfalseIf present, animation re-triggers and class is removed every time element leaves viewport.
ss-groupeffect name—Applies effect to all direct children with staggered delays.
ss-staggernumber (ms)80Delay between each child in a ss-group.
ss-navclass name—On a <nav> element: automatically adds class to links whose href target section is in view.
ss-progressstring (id)—Fires a named progress event (0–1) as user scrolls through the element.
ss-durationnumber (ms)700Override animation duration via CSS var or this attribute.

JavaScript API

MethodArgs—Description
ScrollSpark.init()options?—Initialize, scan DOM for ss-* attributes, return instance.
ss.on()selector, options—Add a scroll trigger programmatically. Options: animate, delay, threshold, repeat, onEnter, onLeave.
ss.addClass()selector, class, options?—Toggle a CSS class when element enters/leaves viewport.
ss.stagger()selector, options—Animate children in cascade. Options: animate, stagger, delay, threshold, repeat.
ss.progress()selector, callback—Call callback(ratio: 0–1) as user scrolls through element.
ss.trackNav()navSelector, activeClass—Highlight nav links when their target section is visible.
ss.off()selector—Remove triggers from elements.
ss.refresh()——Re-scan DOM (use after dynamic content changes).
ss.destroy()——Disconnect all observers, clean up everything.
